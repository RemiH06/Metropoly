Installation

git clone https://github.com/RemiH06/Metropoly.git
cd Metropoly
pip install -r requirements.txt
Windows

Make sure Visual C++ Redistributable DLLs are installed.

Quick Start

# Generate tiles, cards and board
python generator.py

# Generate fortune cards
python fortunaFactory.py

# Generate rulebook
python instructivoFactory.py

# Assemble complete print directory
python gameFactory.py

generator.py

Main entry point. Loads zmg.csv, builds sorted lane lists, resolves colors, and calls cardFactory and boardFactory.

python generator.py                # smart cache
python generator.py --force        # regenerate everything
python generator.py --workers 3    # parallel image scraping
Smart Cache

Tiles are only regenerated when: the image is missing, @font-face is absent while the TTF exists, or the resolved color changed.

cardFactory.py

Generates tile HTMLs (150×225px, 4 rotations) and property card HTMLs (200×350px) for every row in zmg.csv.

PropiedadData class holding all tile fields from CSV

_TIPO_DETALLEDict mapping tile type → detail rows for the card

Non-buyable types

Types 10 (fortune), 14 (exchange house) and 15 (payday) have precio=0 and never show a purchase price or mortgage value on their cards.

boardFactory.py

Generates the board HTML by inlining all tile HTMLs into a single file. CSS is scoped per tile using a unique prefix to avoid conflicts.

Key functions

SymbolDescription
saveBoardHtml(...)Main function — builds and saves the board HTML
createRingCells(size, lane, corners, ...)Builds one concentric ring of tile cells
iterRingCoordinates(size)Yields (row, col) coordinates in ring order
sideLengthFromPerimeter(p)Converts perimeter tile count to grid side length
CSS Scoping

Each inlined tile gets a unique .s{uid} prefix on all its CSS rules. A brace-depth parser handles nested rules and base64 content without breaking.

fortunaFactory.py

Generates horizontal fortune cards (350×200px) from fortunas.csv. Uses src/gw/gw_{lane}{level}.png sprites; falls back to a colored placeholder if sprites are missing.

python fortunaFactory.py
python fortunaFactory.py --force
python fortunaFactory.py --input props/fortunas.csv

colorResolver.py

Resolves color=auto values and provides the group color system. Blue lane type-1 properties are sorted by price and assigned to groups of 3–4 tiles per color.

Guardian cards

Kept in hand and activated when the player chooses. Level 3+ guardian cards are lost when a player goes to jail.

Color System

Colors are resolved in this priority order:

1

Explicit color in CSV

If color is not auto, it's used directly.

2

Type fixed color

Types like trains, airports, casinos always use a fixed color regardless of position.

3

Price group (blue type-1 only)

Properties sorted by price are assigned to groups of 3–4, each with a unique color from the 13-color palette.

4

Fallback

Unresolved tiles fall back to blue (lane default).

Optional: Image Scraper

When --workers is passed, the generator scrapes images for each tile using Selenium. Results are cached in src/img/{tile_name}/.

Requirements

Chrome + Selenium ≥ 4.10.0. ChromeDriver from github.com/dreamshao/chromedriver placed in webdriver/.

python generator.py --workers 3

Three strategies are attempted per tile: thumbnail click → <img> extraction → page source parse. The scraper skips tiles that already have a cached image.
